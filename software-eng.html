    software quality assurance (SQA) charged with overseeing and enforcing the quality control systems adopted by the organization.
    software testing
  • glass box testing : the software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
  • black box testing : tests that do not rely on knowledge of the software's interior composition.
  • beta testing : a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
  • alpha testing : Similar testing performed at the developer's site.

    Documentation
  • user documentation : to explain the features of the software and describe how to use them.
  • system documentation : to descibe the software's internal composition so that the software can be maintained later in its life cycle.
  • technical documentation : to describe how a software system should be installed and serviced.

    intellectual property law
  • software license : legal agreement between the owner and user of a software product that grants the user certain permissions to use the products without transferring ownership rights to the intellectual property.
  • software disclaimer : paintiff can show negligence on the part of the defendant.
  • best defenses against software liability claims : to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
